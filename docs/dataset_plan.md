# 数据集到任务的设计说明

## 1) LOL（低照度增强）

- 输入：`low`
- 标签：`high`
- 任务：`low_light`
- 目标：提升亮度、对比度与色彩真实性

## 2) SIDD（真实噪声去噪）

- 目录：`data/SIDD_Small_sRGB_Only/Data`
- 任务：`denoise`
- 当前基线：从干净图合成轻噪声进行监督训练（后续可替换为官方配对规则）

## 3) DIV2K（超分辨率）

- 输入：由 HR 在线下采样构造 LR
- 标签：HR 原图
- 任务：`super_resolution`
- 当前比例：`x2`

## 4) Places365（场景修复/补全）

- 输入：随机不规则掩膜后的图像
- 标签：完整原图
- 任务：`inpainting`
- 掩膜：`random_irregular`

## 5) faces_dataset_small（人脸修复）

- 输入：通过模糊+噪声退化的人脸图像
- 标签：原始人脸图像
- 任务：`face_restoration`

---

## 多任务训练策略建议（毕业设计可写）

- **阶段一（基线）**：单任务分别训练，得到每个任务基线结果。
- **阶段二（统一模型）**：共享主干 + 任务条件编码（task embedding / adapter）。
- **阶段三（对比实验）**：
  - 单任务模型 vs 多任务共享模型
  - 不同损失函数组合（L1 / SSIM / 感知损失）
  - 不同任务采样策略（均匀采样 / 温度采样）

## 指标建议

- 通用：PSNR、SSIM
- 感知质量：LPIPS（可选）
- 主观可视化：每任务至少展示 4 组输入-输出-真值对比图
